全局配置

可以用 config 方法对 SeaJS 进行全局配置： 

seajs.config({
  base: 'path/to/jslib/',
  alias: {
    'app': 'path/to/app/'
  },
  map: [
    ['http://a.tbcdn.cn/', 'http://localhost/']
  ],
  charset: 'utf-8',
  timeout: 20000,
  debug: false
});
  • base - top-level id 的相对目录，默认指向 sea.js 所在目录。
  • alias - 全局别名配置。
  • map - 文件路径映射表。
  • charset - 下载文件时 script 元素的 charset 属性。
  • timeout - 下载文件的最大时长。
  • debug - 0: 关闭调试； 1: 打开调试； 2: 打开调试并自动添加时间戳。

alias

当模块标识很长时，可以通过 alias 设置来简化模块标识：

seajs.config({
  alias: {
    'app': 'path/to/app',
    'jquery': 'jquery/1.6.1/jquery'
  }
});

/** a.js **/
define(function(require, exports, module) {
    var $ = require('jquery');
    // 解析后：require('jquery/1.6.1/jquery');

    var biz = require('app/biz');
    // 解析后：require('path/to/app/biz');
});

noConflict

为了避免冲突，或期望能自定义全局变量的命名时，可通过 noConflict 方法来实现：

var myLoader = seajs.noConflict();
myLoader.use('./main');

/* main.js */
define(function(require, exports, module) {
  // snip...
});

通过 true 参数，还可以避免 define 方法的冲突：

var myLoader = seajs.noConflict(true);
myLoader.use('./main');

/* main.js */
myLoader.define(function(require, exports, module) {
  // snip...
});
