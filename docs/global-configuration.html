全局配置

可以用 config 方法对 SeaJS 进行全局配置： 

seajs.config({
  base: 'path/to/jslib/',
  alias: {
    'app': 'path/to/app/'
  },
  charset: 'utf-8',
  timeout: 20000,
  debug: false
});
  • base - top-level id 的相对目录，默认指向 sea.js 所在目录。
  • alias - 全局别名配置。
  • charset - 下载文件时 script 元素的 charset 属性。
  • timeout - 下载文件的最大时长。
  • debug - 调试信息的开关。

alias

当模块标识很长时，可以通过 alias 设置来简化模块标识：

seajs.config({
  alias: {
    'app': 'path/to/app'
  }
});

/** a.js **/
define(function(require, exports, module) {
    var b = require('app/b');
    // 等价于：var b = require('path/to/app/b');
});

还可以通过 alias 方法来快捷设置别名：

seajs.alias('app', 'path/to/app');

alias 方法支持正则：

seajs.alias('([^/]+)$', '$1-v2');
// 将 require('./abc') 等价成 require('./abc-v2')
