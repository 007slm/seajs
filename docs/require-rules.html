require 规则

在 SeaJS 里，当对模块进行打包部署时，要分析模块的依赖关系。为了保证依赖分析的正确性，在书写模块代码时，要遵守以下规则：

1. 参数名必须是 require .

// 错误!
define(function(req) {
  // ...
});

// 正确!
define(function(require) {
  // ...
});

2. 不要对 require 进行任何修改。

// 错误 - Renaming "require"!
var req = require, mod = req("./mod");

// 错误 - Redefining "require"!
require = function() {};

// 错误 - Redefining "require" as a function parameter!
function F(require) {}

// 错误 - Redefining "require" in a nested scope!
function F() {
    var require = function() {};
}

3. require 仅接收字符串字面量。

// 错误!
require(myModule);

// 错误!
require("my-" + "module");

// 错误!
require("MY-MODULE".toLowerCase());

// 正确!
require("my-module");

注：SeaJS 的以上规则和 FlyScript 一致： Static Analysis
